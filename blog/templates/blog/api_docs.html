API Documention

1.1 URL's

To use the DjangoBooks.API, go to the links in the following table

Link Name Description Format
/api API Total information dictionary JSON
/api/authors API/Authors Information dictionary about authors JSON
/api/authors/(id of the author) API/Authors/Single Information dictionary about a single author JSON
/api/books API/Books Information dictionary about books JSON
/api/books/(id of the book) API/Books/Single Information dictionary about a single books JSON
/api/categories API/Categories Information dictionary about a categories JSON

1.2 Handling JSON Response

The website returns a JSON dictionary if you request an item. Here is a little example of how it returns the total dict:

{"books": {"Les Miserables": {"id": 2, "author": "Victor Hugo", "url": "/api/books/2/", "download_link": "/media/books_storage/Les-Miserables.pdf", "category": "Epic novel", "tags": ["Les Miserables", "Epic", "Historical Fiction", "Victor Hugo"]}, "Moby Dick": {"id": 3, "author": "Herman Melville", "url": "/api/books/3/", "download_link": "/media/books_storage/Moby-Dick.pdf", "category": "Adventure Fiction", "tags": ["Moby", "Herman Melville", "Adventure", "Fiction", "Dick"]}, "The Hound of the Baskervilles": {"id": 1, "author": "Sir Arthur Conan Doyle", "url": "/api/books/1/", "download_link": "/media/books_storage/TheHoundoftheBaskervillesDB.epub", "category": "Detective", "tags": ["Detective", "Sherlock Holmes", "Hound", "Dog", "England"]}}, "authors": {"Sir Arthur Conan Doyle": {"id": 1, "url": "/api/authors/1"}, "Rafayel": {"id": 4, "url": "/api/authors/4"}, "Victor Hugo": {"id": 5, "url": "/api/authors/5"}, "Herman Melville": {"id": 6, "url": "/api/authors/6"}}, "categories": {"Detective": {"id": 3, "author": "Sir Arthur Conan Doyle"}, "Epic novel": {"id": 4, "author": "Rafayel"}, "Adventure Fiction": {"id": 5, "author": "Rafayel"}}}

It returned now the books, authors and the categories but there are much more thing that are coming soon! To handle this response you can use the standard methods of your PL, because it returns data in JSON format

The standard /api returns first books dictionary, then the authors dictionary and then the categories dictionary
